# 贡献说明

提交代码或文档前，请确保：

- 不包含密码、私钥、签名文件或个人配置。
- 不复制来源不明或许可证不兼容的代码与资源。
- 不把规划中的功能描述成已经实现。
- 提交说明能够准确描述改动目的。
- 新增能力应保留清晰的边界，避免把页面、分析、存储和系统服务逻辑堆进单一文件。

## 文档贡献指南

本文档目录为 `docs/`，所有文档统一使用中文书写。

### 已实现与规划必须分离

在 README、CHANGELOG 或任何公开文档中，必须严格区分：

- **已实现**：代码仓库中已有对应源码、编译通过并可被实际调用的能力。
- **规划中**：仅在 README 路线图或设计文档中提及，尚未有对应实现的能力。

不允许将以下内容描述为已完成：

- 屏幕采集（MediaProjection）
- 实时视觉识别
- 实时 HUD 绘制
- 视频回放分析
- 糖果/分数识别
- 战报存储

### 贡献流程

1. 修改代码前，先阅读 `docs/architecture.md` 了解当前模块边界。
2. 新增能力时，同步更新 `docs/analysis-algorithm.md` 中的算法描述。
3. 涉及公共行为变更时，在 `CHANGELOG.md` 中添加条目。
4. 提交前执行 `git diff --check` 检查行尾空白。

### 隐私与安全红线

禁止在代码、文档或 Issue 中包含以下内容：

- 签名密钥、密码或证书文件
- 用户个人身份信息（姓名、手机号、地址）
- 未经脱敏的设备日志或屏幕截图
- 任何第三方平台的账号凭证

---

如需了解项目边界与隐私政策，请参阅 [docs/privacy-and-boundaries.md](docs/privacy-and-boundaries.md)。
