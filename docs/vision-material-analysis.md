## 2. 场景分析结论

### 2.1 地面跑酷场景（主要分析目标）

**特征**：

- 玩家角色位于屏幕中下部（Y 归一化坐标约 0.66 ~ 0.84）
- 游戏世界整体向左滚动，背景滚动速度约 -0.45 归一化坐标/秒
- 危险物从屏幕右侧进入，向左侧移动
- 地面/平台通常位于屏幕下半部分（Y > 0.7）

**结论**：

- 玩家矩形归一化尺寸约为宽 0.10、高 0.18（相对整个屏幕）
- 蛋糕断层宽度通常在 0.05 ~ 0.25 归一化坐标之间
- 宽断层（需要二连跳）宽度 >= 0.18 归一化坐标
- 裱花袋悬挂在屏幕顶部区域（Y < 0.35）

### 2.2 飞行模式场景

**特征**：

- 玩家角色在空中飞行，无地面参照
- 背景滚动速度可能不同于地面模式
- 不应输出跳跃/滑铲提示

**结论**：

- 飞行模式的视觉特征与地面模式差异明显（无地面/平台纹理）
- 场景状态机 `SceneStateMachine` 需正确识别并切换至 `kFlightRun`

### 2.3 遮挡场景

**特征**：

- 通知栏、来电、弹窗等外部元素覆盖游戏画面
- 画面中出现非游戏 UI 元素

**结论**：

- 遮挡检测阈值设为 `kOcclusionThreshold = 0.72`
- 遮挡状态下提示置信度阈值提升至 0.85

## 4. 分辨率适配结论

### 4.1 归一化坐标有效性验证

所有素材经抽帧验证，游戏内容在归一化坐标系（0.0 ~ 1.0）下表现一致：

| 分辨率 | 玩家 Y 位置 | 地面 Y 位置 | 危险物 Y 位置 | 结论 |
| --- | --- | --- | --- | --- |
| 570x1280 | ~0.75 | ~0.78 | ~0.78~0.98 | 一致 |
| 1080x2400 | ~0.75 | ~0.78 | ~0.78~0.98 | 一致 |
| 1746x3840 | ~0.75 | ~0.78 | ~0.78~0.98 | 一致 |

**结论**：归一化坐标系在不同分辨率下具有良好的一致性，算法参数无需按分辨率分段设定。

### 4.2 像素级面积阈值的分辨率依赖

虽然归一化坐标一致，但**绝对像素面积**随分辨率变化：

| 分辨率 | 玩家矩形像素面积（约） | 毒瓶像素面积（约） |
| --- | --- | --- |
| 570x1280 | ~570 x 260 = 148K px | ~30 x 60 = 1800 px |
| 1080x2400 | ~108 x 216 = 23328 px | ~12 x 24 = 288 px |
| 1746x3840 | ~175 x 180 = 31500 px | ~15 x 30 = 450 px |

**结论**：形态学处理的面积阈值（`min_contour_area`、`max_contour_area`）需要按分辨率档位分别设定，或使用归一化面积（相对于屏幕总面积的比例）。

## 5. 帧率与时间分析

### 5.1 素材帧率

| 素材 | 帧率 | 帧间隔 |
| --- | --- | --- |
| 大部分素材 | 60 fps | 16.67 ms |
| 部分高分辨率素材 | 90 fps | 11.11 ms |

### 5.2 对算法的影响

- C++ 分析引擎的垂直速度计算基于帧间隔（`delta_seconds`），不同帧率下计算结果自然适配。
- 场景状态机的"连续两帧确认"机制不依赖时间戳，仅依赖帧计数，因此 60fps 和 90fps 素材的行为一致。
- 二段跳检测的最小脉冲间隔 `kMinImpulseGapMs = 95ms` 在 60fps 下约 6 帧、在 90fps 下约 9 帧，均能有效防止重复触发。
