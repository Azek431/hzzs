# 隐私与功能边界

本文档描述 HZZS 当前主分支已经存在的能力、数据处理方式和安全边界。功能变化时必须同步更新，不能把已经实现的敏感能力写成“尚未实现”，也不能把规划中的保护措施描述成已经生效。

## 核心定位

HZZS 是本地运行的跑酷画面分析与 Canvas HUD 辅助工具，包含需要用户明确开启的实验性无障碍手势能力。

- 画面分析和动作规划在设备本地完成。
- 应用本身未申请 INTERNET 权限，不包含向项目服务器上传画面、配置或日志的实现。
- QQ、Telegram 等社区入口会把用户交给外部应用或浏览器，离开 HZZS 后适用对应第三方的隐私规则。
- 屏幕采集、无障碍服务和悬浮窗都需要用户通过 Android 系统入口授权。

“本地运行”不等于“没有敏感能力”。屏幕内容和无障碍手势都属于高风险能力，必须遵守下文的授权和失败关闭规则。

## 当前已实现能力

| 能力 | 当前实现与边界 |
| --- | --- |
| 屏幕采集 | 支持 Android 11+ 无障碍截图、用户授权的 MediaProjection，以及只能手动选择的实验性 Root 模式 |
| 实时视觉分析 | 正式 vision2 核心可分析绿瓶、蛋糕结构和悬挂尖刺；当前每一类只返回一个候选目标 |
| Canvas HUD | runtime HUD 可显示分析状态、检测框、距离和动作状态；仓库中仍有待删除的旧模拟 HUD |
| 可选自动操作 | 无障碍服务可注入跳跃和下滑手势；首次安装和安全迁移后默认关闭 |
| 前台目标限制 | 只有可信窗口切换事件记录的允许包名才能消费动作，目标未知或不允许时清空队列 |
| 本地设置与日志 | 配置保存在 SharedPreferences；日志和诊断主要用于本地排错 |
| 社区入口 | 通过系统 Intent 打开 QQ 或 Telegram，不由 HZZS 内置网络客户端访问 |

## 当前仍需补强的边界

以下内容是正在改造的缺口，不应被描述为已经完成：

- 前台目标状态尚未具备短时有效期和目标会话令牌。
- 旧动作队列与正式 runtime 动作队列仍同时存在。
- 自动操作授权仍分散在两份 SharedPreferences 中。
- 三套 Overlay 和三条原生视觉链路尚未完全收敛。
- 采集链路仍会产生不必要的全屏 Bitmap 分配和复制。
- 尚未建立统一 DataStore、配置导入安全策略和版本化同意记录。

在这些保护完成前，自动操作只能视为默认关闭的实验能力。

## 授权与失败关闭

### 屏幕采集

- MediaProjection 每次由用户通过系统授权界面同意，服务停止或系统撤销后不能继续使用旧授权。
- 无障碍截图只在用户主动启用 HZZS 无障碍服务后可用。
- Root 模式不会被 AUTO 模式自动选中，只能由用户主动选择，并取决于设备自身的 Root 授权。
- runtime 当前只消费内存中的帧，不以正常功能为由自动保存完整截图或视频。

### 自动操作

- 自动操作首次安装默认关闭。
- 历史版本留下的开启值会经过安全迁移强制关闭一次。
- 服务未连接、前台目标未知或不在允许列表时不得执行手势。
- 停止 runtime、关闭自动操作或断开无障碍服务时会清空动作队列。
- 外部配置、备份恢复和设备迁移不得静默打开自动操作。

当前备份规则会排除 hzzs_feature_flags.xml 和 hzzs_runtime_v2.xml。这样会同时放弃恢复其中的普通设置，但可以保证免责声明记录和自动操作授权不会跨设备静默继承。统一配置上线后，应把安全授权和同意记录迁入独立存储并继续排除。

## 数据处理原则

### 不上传

- 应用未实现远程分析、遥测、广告或账号系统。
- 不主动上传截图、检测结果、动作记录、设备标识或配置。
- 后续若增加任何网络能力，必须先更新权限、界面说明和本文档，不能沿用当前“不上传”的结论。

### 最少留存

- 实时采集帧只应在内存中短暂存在，消费后及时释放。
- 诊断目录、截图目录和视觉帧目录已排除在系统云备份与设备迁移之外。
- 不申请全盘存储权限。
- 将来如增加日志或配置导出，必须由用户主动发起并明确目标位置。

### 日志并非天然匿名

Android 日志可能包含目标包名、运行状态、异常类型和设备环境信息。向 Issue、Pull Request、群聊或第三方发送前必须人工检查和脱敏，不能承诺诊断日志绝不包含隐私信息。

## 明确不提供

| 项目 | 边界 |
| --- | --- |
| 静默采集 | 不在缺少系统授权或明确状态提示时采集屏幕 |
| 远程控制 | 不提供服务器下发动作、远程代打或无人值守控制 |
| 绕过限制 | 不提供绕过系统权限、反作弊、付费或平台限制的能力 |
| 默认自动化 | 不在首次安装、配置导入、备份恢复或设备迁移后自动启用手势 |
| 默认上传 | 不主动上传画面、日志或用户配置 |
| 全盘文件访问 | 不申请管理外部存储或传统全盘写入权限 |

## Issue 与日志提交注意事项

在 GitHub、QQ、Telegram 或其他渠道提交问题时，请不要包含：

- 手机号、邮箱和账号信息；
- 游戏账号、好友列表、聊天记录和支付信息；
- 身份证明、住址等个人敏感信息；
- 包含个人内容的完整屏幕截图或录屏；
- 签名文件、密钥库、密码、令牌或私钥；
- 未脱敏的系统日志。

需要分享日志时，应先把敏感值替换为 REDACTED，并只保留复现问题所需的最小片段。
