# 安全模型

HZZS 会接触屏幕内容、悬浮窗和无障碍手势，这三项能力默认均采用失败关闭策略。

## 自动操作

- 默认关闭，不从备份、配置导入或迁移中恢复开启状态。
- 每次运行会话需要用户重新授权。
- 只有一个 `GestureArbiter` 可以调用 `dispatchGesture()`。
- 前一手势完成或取消前不会提交下一手势。
- 只有系统回调 `onCompleted` 后才将目标轨迹标记为完成。
- 玩家后方、诊断目标、非法类型、过期帧和低置信度结果不能进入动作链。
- 包名、窗口状态、场景置信度和动态玩家检测共同构成动作门控。

## 更新

- Gitee 为国内首选源，GitHub 为备用与交叉校验源。
- APK、差分包和更新清单必须通过 SHA-256、包名、版本号和签名证书校验。
- 差分还原失败时回退完整 APK。
- 安装始终交给 Android 系统界面确认，不支持静默安装。
- 正式密钥和密码只通过本地环境变量或 GitHub Secrets 注入，不进入仓库。
